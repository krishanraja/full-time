# 05 · Content Safety

**Role:** Developer, product, legal, anyone touching the AI pipeline or considering loosening a constraint.
**Read this when:** changing the system prompt, the banned-terms regex, the retry policy, or the AI disclosure copy.
**Don't read this when:** you only need user-facing FAQ (→ `10-support.md`).

---

## Why this matters

We narrate football. Football media is full of three things we have to never do:

1. **Hallucinated facts** — invented goals, scorers, minutes, quotes.
2. **Real-broadcaster impressions** — imitating named commentators is an IP and trust hazard.
3. **Adjacent harms** — transfer rumours, injury speculation, betting language, political takes, slurs.

Any of these in production is a brand-ending event. Treat this file as the binding constraint, not a guideline.

## System prompt (canonical text)

Lives in `src/lib/api/episode-pipeline.functions.ts` as `SYSTEM_PROMPT`. Reproduced here so changes are reviewed:

```
You write 60-second football match recaps (120-150 words) for the "Full Time" daily podcast.
Style rules:
- Confident, lean prose. Short sentences. One vivid image per paragraph.
- ONLY use facts provided. Do not invent goals, scorers, minutes, or quotes.
- No transfer rumours, no injury speculation, no betting language.
- No impressions of real broadcasters. No political commentary. No slurs.
- Always end with the final score stated naturally.
Output the recap text only — no titles, no preambles, no markdown.
```

**Rule:** any change to this prompt requires:
1. Updating this file in the same change.
2. Logging a decision in `12-roadmap.md`.

## Inputs we pass to the model

Only the structured match fact set:

- Home team name, away team name, competition name, home score, away score.

We do **not** pass: rumours, betting lines, player quotes, news headlines, prior commentary, our own past episodes.

## Banned-terms filter

Lives in `src/lib/api/episode-pipeline.functions.ts` as `BANNED_TERMS` regex. Current pattern (canonical here):

```
/\b(bet|odds|over\/under|injur(?:y|ed)? (?:rumour|rumor)|transfer rumou?r|allegedly|sources say|i hate|stupid|moron|idiot|f[*u]ck|sh[*i]t)\b/i
```

Workflow:
1. Generate script.
2. If `BANNED_TERMS.test(script)` → throw, caught by `generateEpisodeForMatch` outer try, **one retry**.
3. If the retry also fails → throw out of the per-match pipeline. The cron logs the failure and **does not publish** that match.

Add a term by editing the regex AND adding a row here:

| Term | Added | Why |
|---|---|---|
| `bet`, `odds`, `over/under` | v1 | Betting language is never used. |
| `injury rumour`, `transfer rumour`, `allegedly`, `sources say` | v1 | We don't speculate. |
| `i hate`, `stupid`, `moron`, `idiot` | v1 | No personal attacks. |
| profanity stems (`f*u*k`, `sh*i*t`) | v1 | Family-safe by default. |

## What we don't filter (deliberately)

- Player names. We name scorers as the source data names them.
- Sharp language about performance ("collapsed", "outplayed", "ragged"). Match reporting needs colour.
- Manager names and tactical critique grounded in the score line.

## Voice / TTS constraints

- One voice per `voice_style`. No cloning of real broadcasters. Voice IDs live in the pipeline file; documented changes go in `12-roadmap.md`.
- Output format: `mp3_44100_128`. Don't change without product sign-off — affects file size and bandwidth cost.

## AI disclosure (user-facing)

The Settings page surfaces:

> Recaps on Full Time are generated by AI from publicly available match data. Voices are synthetic. No copyrighted broadcast audio is used.

Every player surface shows a low-volume `AI · {duration}` tag. Do not hide either. See `11-legal.md` for the legal stance.

## What to do if something slips through

1. Take the offending episode down immediately: delete the `episodes` row and the Storage object at `episodes/YYYY-MM-DD/{matchId}.mp3`. Runbook in `06-ops.md`.
2. Log the script that produced it in an internal note.
3. If it was a regex miss → add the term to `BANNED_TERMS` in the same change.
4. If it was a hallucinated fact → check the input — the source data may be wrong, or the model may need a tighter prompt.
5. Post a short note on Settings / changelog if the fix is user-visible.

## Things we will not do, even if asked

- "Make it funnier" by loosening the banned-terms list.
- Use real broadcaster voices, even with a licence claim, without legal sign-off.
- Include "what's the betting" type asides.
- Pipe in rumour news from any RSS / aggregator.

If a stakeholder asks for any of the above, point them at this file.
